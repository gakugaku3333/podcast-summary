📌 Claude デスクトップアプリと「Computer Use」

  • ネイティブアプリの操作が可能に
    • ブラウザ経由ではなく、PCにインストールされたCapCutなどのネイティブアプリをClaudeが直接操作できるようになった。
    • 「URL上で触れる」レベルから「PC全体の操作」へ幅が広がった。
  • セキュリティとリスク
    • 情報漏洩のリスクは常に存在する。特にClaude CodeやCo-arcでは、意図しないデータ公開に注意が必要。
    • ただし、ブラウザ操作が許可されている現状と比べ、デスクトップアプリ化によるリスクの激増はないという見解。
  • ユースケースの具体化が課題
    • 「何でもできる」がゆえに、具体的な使い道がイメージしづらい初期AIの状態に近い。
    • 早川ゴミ氏の提案：3Dプリントのセッティング自動化。深夜に印刷が終わった後の次工程のセットなどをAIに任せる可能性。

📌 Claudeの驚異的な開発スピード

  • 52日間で73個のアップデートを実行したというデータがある。
  • UIを設計しない戦略：Claude CodeなどはCLI（コマンドライン）ベースのため、UI開発に時間を取られず、内部コードの改善のみで高速にリリースできる。

📌 ChatGPTからの「知識移植」トレンド

  • 「私について知っていることを全部教えて」プロンプト
    • ChatGPTに蓄積されたユーザーのコンテキスト（メモリ）を長文で書き出させ、それをClaudeやGeminiのカスタムインストラクションに移植する手法。
    • 「ChatGPTにデータが溜まっているから乗り換えられない」というロックイン効果を打破する動き。
  • AIの「記憶」に関する課題
    • AIが何でも覚えすぎてしまい、関係ない文脈で過去の情報を出してくる（例：ロボット製作中に過去の長野旅行の話を混ぜる）。
    • 人間のように「忘れること」や「情報の重み付け」を制御する機能が今後重要になる。

知識移植のフロー：ChatGPTから情報を抽出し、他AIへ再学習・設定させる

📌 Gemini Live の高速化とエンタメ展開

  • 3.1 Flash対応による低遅延化
    • 会話のレスポンスが大幅に向上。リアルタイムでの画面共有・解説がよりスムーズに。
  • ドラゴンクエストとのコラボ
    • ゲーム内にGeminiカラー（Googleカラー）のスライムが登場。

  • クリエイティブ界隈のAIアレルギー
    • ミュージックビデオやプロ野球のカウントダウン画像でのAI使用が批判されるケースも多い。
    • 「数年経てば当たり前になる」という過渡期の現象。

📌 「AI力検定」の構想（うすたく氏）

  • 知識問題ではなく「言語化力」を問う
    • 既存の試験は「スマホ禁止の知識暗記」が多く、実務に即していない。
    • 「画面に映っているものを正確に言葉で説明できるか」といった、AIに指示を出すための基礎能力を測定すべき。
  • 試験形式のアイデア
    • TinderのようなUIで、2秒以内に判断してスワイプする形式（AIを使う暇を与えない速度で基礎知識を問う）。
    • 「SonnetとHaiku、どちらが速いか？」といった、実務者が体感で知っておくべき知識をフラッシュカード的に問う。
