📌 いけとも氏：Vision KPI ダッシュボードとLocal LLM

  • 「勝てるゲーム」の設計：
    • 売上や再生回数といった他人の軸ではなく、自分がテンションの上がる「累積（ストック）」を指標にする。
    • イベントアンケート、本のレビュー、YouTubeコメントなど、あらゆる「価値創出の事例」をデータベース化。
  • Claude Designによる模索：
    • 「どういう見せ方なら自分のテンションが上がるか」をClaude Designでプロトタイピングし、ダッシュボードのUIを決定。
  • Local LLM（Gemma 2）の活用：
    • メールアドレスを含む個人情報や、深い関与データ（Xの発信内容など）を処理するため、Mac上のローカル環境でLLMを回す。
    • 外部APIコストを気にせず、24時間体制で「この人は今何を求めているか」を分析・レベル定義する体制を構築。

📌 尾原氏：Live Dashboardとスキルのエージェント化

  • Live Dashboard（ライブダッシュボード）：
    • 「会議のための資料」を廃止し、日常的に使うダッシュボードをそのまま議論のベースにする。
    • Vibe Codingにより、会議中のイシューに合わせてその場でダッシュボードを書き換える。
  • スキルの構造化と共有：
    • 単なるテキスト（Markdown）の共有から、Claude.mdやSkill.mdといった「エージェント定義ファイル」の共有へ進化。
    • 他人の優れたスキル（例：オンボーディングのスキル）をリバースエンジニアリングし、自分のプロジェクトに移植する「技マシン」的な運用。
  • ハイパー・パーソナライゼーション：
    • AIにより、既存顧客一人ひとりの成長度合いに合わせたCRM（顧客関係管理）がようやく実装可能になった。

📌 AI時代の本質的な変化

  • 「燃え尽き」への対策：
    • AIの進化が速すぎて人間が置いていかれる感覚（他人事化）を防ぐため、いかに「自分を生き生きとさせ続けるか」というパーソナライゼーションが重要。
  • ステップバックの自動化：
    • 人間は目の前の業務に埋没しがちだが、AIに「一歩引いた視点（スコープの拡大）」や「構造化」を壁打ちさせることで、持続的な競争優位性を保つ。
  • ホスピタリティのブランド化：
    • 深いデータ分析に基づく支援が「気持ち悪いストーカー」ではなく「リッツカールトンのようなホスピタリティ」として受け取られる信頼関係（オーセンティシティ）の構築。
「単なるツールとしての深掘りではなく、ツールが進化したからこそ、もともとやりたかったことがどう実現できるのかを考えるのが楽しい。」
