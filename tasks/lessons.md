# 📋 開発の教訓 (Lessons Learned) — 2026/03/16

今回の「Podcast 自動解説」の開発、および GitHub Pages 連携機能の実装を通じて得られた教訓をまとめます。

## 🛡 トラブル回避のポイント

### 1. Git の巨大ファイル問題 (GH001)
- **問題**: `venv` 内の ML ライブラリ（torch, mlx 等）が 100MB を超え、プッシュが拒否された。
- **原因**: 最初の `git add .` の前に `.gitignore` で `venv/` を除外し忘れていた。一度 Git の履歴に入ると、ファイルを消しても履歴データが残るため、プッシュが通らなくなる。
- **教訓**: プロジェクト開始直後に `.gitignore` を設定すること。万が一巨大ファイルが混じった場合は、履歴の削除（`rm -rf .git` での再初期化）が最も確実で早い。

### 2. ポートの競合 (Address already in use)
- **問題**: 手動起動プロセスと launchd プロセスが同時に動こうとしてポート 8000 で衝突した。
- **教訓**: `launchd` や `systemd` で管理するサービスを修正する際は、必ず `lsof -ti:<port> | xargs kill -9` で古いプロセスを掃除してからリロードすること。

### 3. ステータスの固着 (Frozen Status)
- **問題**: 3/4 に起きたクラッシュにより、特定のエピソードが `transcribing` のまま 10日間以上止まっていた。
- **教訓**: 処理中のエピソードが一定時間を超えて完了しない場合に、監視スクリプトでステータスを自動的に `pending` にリセットするような自己回復機能を検討すること。

### 4. 自動プッシュのハングアップ
- **問題**: `git push` が認証待ちで入力を求めて停止し、サーバー全体がブロックされる。
- **解決**: `GIT_TERMINAL_PROMPT=0` 環境変数を設定することで、対話的な入力を即座にエラーとして終了させ、サーバーのフリーズを防ぐことができる。

## ✨ 機能活用のためのポイント

### GitHub Pages 連携
- **メリット**: 静的な HTML を `docs/` に置くだけで、Mac をインターネットに直接さらす（Cloudflare Tunnel 等）ことなく、安全かつ無料で要約を全世界（または自分だけ）に公開できる。
- **Jekyll の無効化**: `docs/.nojekyll` を置くことで、GitHub 側の不要なビルド処理（Jekyll）をスキップし、生の HTML/CSS がそのまま表示されるようにした。
